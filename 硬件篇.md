# **硬件篇**

## **1.逻辑电平（电平值不一定准确，自行验证）**

| 符号 | 含义                                    | 一般要求                                   |
| ---- | --------------------------------------- | ------------------------------------------ |
| VIH  | 输入高电平最小值（Voltage Input High）  | 输入高电平 ≥ VIH 时，器件保证识别为逻辑“1” |
| VIL  | 输入低电平最大值（Voltage Input Low）   | 输入低电平 ≤ VIL 时，器件保证识别为逻辑“0” |
| VOH  | 输出高电平最小值（Voltage Output High） | 输出高电平 ≥ VOH 时，被下一级识别为“1”     |
| VOL  | 输出低电平最大值（Voltage Output Low）  | 输出低电平 ≤ VOL 时，被下一级识别为“0”     |

一般来说VOH\>VIH\>VIL\>VOL,以此来保证输出级能够包容输入级

## **2.阻抗匹配**

单端50Ω（或者100Ω），差分100Ω

阻抗匹配主要是为了抑制反射

1.**示波器探头（单端探测）**

-   **无源探头**：最常见的是 **10:1 探头**，输入阻抗典型是 **10 MΩ // 10\~20 pF**。
-   **有源探头（单端）**：为了带宽高、减小负载，输入阻抗一般是 **1 MΩ // 几 pF**，输出是 50Ω 匹配示波器。

2.**传输线 / 信号接口的特性阻抗**

-   **同轴电缆（单端传输）**：常见有两种标准：

**50Ω**：主流，高速数字信号、射频、以太网 PHY（如 SMA 接口）

**75Ω**：视频、电视、广播（同轴电视线、摄像头 BNC）

-   **差分传输线**（比如 PCB 上的差分对）：

常见是 **100Ω 差分阻抗**（即每条线对地约 50Ω，但作为差分对时是 100Ω）。

PCIe、LVDS、USB、Ethernet 都是 100Ω 差分。

## **3.逻辑门**

| 门类型     | 输出类型        | 未使用时需要上拉/下拉吗？ | 说明                             |
| ---------- | --------------- | ------------------------- | -------------------------------- |
| 普通逻辑门 | 推挽            | 可选（输入未使用）        | 输出有驱动，不悬空               |
| OE 门      | 推挽 + 高阻控制 | 建议上拉/下拉             | 确保默认输出状态安全             |
| OC/OD 门   | 开路 / 高阻     | 必须上拉                  | 高电平由上拉提供，否则悬空不确定 |

**常见输出类型**

### **3.1 普通推挽输出 (Totem-pole Output)**

-   **结构**：上下两个晶体管（P/N 或 NPN/PNP）组成，可输出高电平或低电平。
-   **特点**：电平驱动能力强，但**不允许多个输出直接并联**，否则会短路。
-   **示例**：标准 TTL、CMOS 门电路。

### **3.2 三态输出（OE 门，Output Enable）**

-   **特点**：除高电平、低电平外，还有 **高阻态 (Hi-Z)**。
-   **作用**：通过 OE 控制引脚决定是否驱动总线，多个器件可共享一条数据线（总线结构）。
-   **示例**：74HC245 总线收发器，存储器数据引脚。

### **3.3 开集电极输出（OC 门，Open Collector）**

-   **结构**：输出端只有一个集电极接出，内部没有上拉电阻。
-   **特点**：

只能下拉（拉低），不能输出高电平，需要外部上拉电阻到 VCC。

多个 OC 输出可以并联到同一线上，形成 **线与 (Wired-AND)**：

所有输出均释放（上拉）→ 线为高电平

任一输出拉低 → 线为低电平

-   **示例**：I²C SDA/SCL、一些中断信号。

### **3.4 开漏极输出（OD 门，Open Drain）**

-   **结构**：MOS 管的漏极开路，本质上和 OC 一样，只是工艺不同（CMOS vs TTL）。
-   **特点**：同 OC 门，需外接上拉电阻，支持线与。
-   **示例**：MCU GPIO 的 OD 模式、I²C。
1.  **线与 (Wired-AND) 的物理情况**

**电路原理**

-   使用 **开集电极 (OC)** 或 **开漏极 (OD)** 输出。
-   输出只有 **下拉管 (NPN 晶体管集电极 / NMOS 漏极)**，高电平靠 **外部上拉电阻** 提供。
-   当多个输出并联到一根线上时：

**任意一个管子导通 → 线被拉低 → 输出=0**

**所有管子关断 → 上拉电阻把线拉高 → 输出=1**

**直观理解**

👉 相当于“**谁拉低，谁赢**”，因此形成 **逻辑与关系**。

**典型支持**

-   TTL/CMOS 的 **开集电极门 (OC)**、**开漏极门 (OD)**
-   MCU GPIO 设置为 OD 模式
-   I²C SDA/SCL 总线、中断线、RESET 线

**不支持的情况**

-   普通推挽输出（Totem-pole），因为它会强驱高电平，如果另一个门强驱低电平，会导致**短路/大电流冲突**。

### **线或 (Wired-OR) 的物理情况**

**电路原理**

-   使用 **开发射极 (Open Emitter)** 或 **开源极 (Open Source)** 输出。
-   输出只有 **上拉管 (PNP 晶体管发射极 / PMOS 源极)**，低电平靠外部下拉电阻提供。
-   多个输出并联到一根线上时：

**任意一个管子导通 → 线被拉高 → 输出=1**

**所有管子关断 → 下拉电阻把线拉低 → 输出=0**

**直观理解**

👉 相当于“**谁拉高，谁赢**”，因此形成 **逻辑或关系**。

**典型支持**

-   早期 **ECL (Emitter-Coupled Logic)** 的 **有线或结构**
-   一些 PMOS/PMOSFET 电路
-   CPLD/FPGA 内部可以配置“有线或”逻辑

**不常见的原因**

-   在 TTL/CMOS 中几乎不用“线或”，因为实现困难（需要上拉管），且 CMOS 工艺中一般用逻辑合成来实现“或”。
-   常用的是 **线与 (wired-AND)**。

**哪些门支持 / 不支持**

| 门类型                    | 推挽输出 | OC/OD 输出 | OE（三态输出） | 线与支持                      | 线或支持              |
| ------------------------- | -------- | ---------- | -------------- | ----------------------------- | --------------------- |
| 普通 AND/OR/NOT 门 (推挽) | ✅        | ❌          | ❌              | ❌（短路风险）                 | ❌                     |
| 开集电极 OC 门            | ❌        | ✅          | ❌              | ✅（外接上拉电阻）             | ❌                     |
| 开漏极 OD 门              | ❌        | ✅          | ❌              | ✅（外接上拉电阻）             | ❌                     |
| 三态门 (OE 门)            | ✅        | ❌          | ✅              | ✅（通过总线“只允许一个驱动”） | ✅（总线上可配置逻辑） |
| ECL / PECL / LVPECL       | ❌        | ❌          | ❌              | ❌                             | ✅（支持有线或）       |

**总结直观记忆**

-   **线与**：低电平优先 → OC/OD 输出 + 上拉电阻
-   **线或**：高电平优先 → 开发射极/开源极（ECL/PMOS） + 下拉电阻
-   **推挽输出**：不能直接并联，必须用三态控制（总线结构）

## **4.功耗分析**

**FPGA 动态功耗来源**

动态功耗主要来源于 **充放电电容**：

Pdynamic=α⋅C⋅V\^\^2⋅f

其中：

-   **α (toggle rate)** → 信号翻转率（0→1 或 1→0 的概率）。
-   **C (capacitance)** → 负载电容（和逻辑资源使用多少相关）。
-   **V (voltage)** → 供电电压。
-   **f (frequency)** → 时钟频率。

静态功耗：漏电流

**漏电流功耗原理(静态功耗)**

-   **漏电流** = 器件内部晶体管在关闭状态下仍有微小电流
-   **漏电流功耗公式**：

Pleak=Ileak×VDDP*{leak} = I*{leak} \\times V_{DD}Pleak=Ileak×VDD

-   主要受以下因素影响：

**温度**：温度越高，漏电流指数增长

**工艺节点**：制程越先进（晶体管越小），漏电流越大

**供电电压**：VDD 越高，功耗越大

**使用的逻辑资源数量**：越多晶体管 → 总漏电流越大

-   **注意**：逻辑切换与时钟频率几乎不影响漏电流

## **5.Verilog层次结构**

| 层次                             | 含义                                                     | VERILOG 表示方式                                         | 示例                                           |
| -------------------------------- | -------------------------------------------------------- | -------------------------------------------------------- | ---------------------------------------------- |
| 系统级 (System-level)            | 最高层次，描述整个系统行为、模块交互，不关心硬件实现细节 | SystemVerilog behavioral/transaction-level，或者 SystemC | 用 TLM 描述一个 CPU + DMA + 外设系统的数据流   |
| 算法级 (Algorithmic-level)       | 描述功能算法行为，侧重计算和数据处理，不关心寄存器或时序 | Verilog behavioral（always, for, case 等）               | 用 always 块描述 FIR 滤波器的加法和乘法过程    |
| RTL 级 (Register-Transfer Level) | 以寄存器和组合逻辑之间的数据传输为单位描述，硬件可综合   | always @(posedge clk)，assign 等                         | 用 always @(posedge clk) 写 8 位计数器或状态机 |
| 门级 (Gate-level)                | 用具体逻辑门和触发器描述，接近电路实现                   | netlist, gate primitives（and, or, flip-flop）           | AND、OR、D触发器组合实现加法器或寄存器         |
| 开关级 (Switch-level)            | 最底层，用晶体管开关行为描述电路                         | MOSFET 建模，transistor-level                            | NMOS/PMOS 组成 CMOS 反相器或 NAND 门           |

**说明**

**层次越高 → 越抽象 → 离物理实现越远**

系统级/算法级：关注功能，不关心电路结构

RTL：关注寄存器、组合逻辑，可综合

门级/开关级：关注具体硬件结构、延迟、功耗

**Verilog 可以描述所有这些层次**，尤其是：

高层次行为描述 → always 块、函数、task

RTL → always @(posedge clk) + assign

门级 → primitives（and/or/not/dff）

开关级 → transistor-level netlist（少用，仿真或晶体管设计时）

## **6.纳米级工艺节点**

“nm 节点”在严格物理意义上，其实常指 **MOSFET 的源漏距离 (Source-to-Drain spacing) 或栅长 (Gate Length, Lg)**，也就是晶体管开关通道的长度。

### **1️⃣ 栅长 (Lg) vs 源漏间距**

**栅长 Lg**：栅极两端之间的距离，也就是控制电流通断的有效通道长度。

**源漏距离**：从源区到漏区的间距（不完全等于栅长，但和栅长紧密相关）。

在早期工艺（90nm 以前），**工艺节点号大致等于栅长或源漏间距**。

### **2️⃣ 为什么现在 nm 不完全对应源漏距离**

FinFET/GAA 结构时代，栅长变短，但源漏之间的物理间距和栅长不再严格一致。

芯片厂商用 nm 主要作为 **工艺代次/密度标识**，而非真实物理尺寸。

### **3️⃣ 影响**

栅长 / 源漏距离越短 → 晶体管开关越快 → 可实现更高频率

同时漏电流增大 → 需要更复杂的功耗管理

所以总结一句话：

**早期 nm 工艺 ≈ 栅长 / 源漏距离，现代 nm 工艺更多是工艺代号和密度指标，不再直接对应物理尺寸。**

## **7.FPGA速度等级**

**什么是 FPGA 的速度等级？**

在 AMD/Xilinx 的器件里（Intel/Altera 类似），通常会有不同的 **速度等级**，用 **-1, -2, -3** 来表示：

**数值越大，速度等级越高**（-3 \> -2 \> -1）。

**速度等级越高，工艺选片更优**，关键路径延迟更小，可以跑更高的时钟频率。

所以：

\-3 比 -2 更“快”，时序裕量更大。

\-2 比 -3 “慢”，时序更差。

**功耗和速度等级的关系**

这跟 **晶体管工艺的阈值电压 Vth** 有关：

**速度快的器件（-3）**：一般采用低 Vth 工艺，晶体管开关更快 → 速度优异，但漏电流大 → **静态功耗更高**。

**速度慢的器件（-2）**：一般采用高 Vth 工艺，晶体管开关慢一些 → 时序差，但漏电流小 → **静态功耗更低**。

换句话说： -3 速度好，但功耗高；-2 速度差，但功耗低。

## **8.FPGA IO**

**基础概念**

**上拉（Pull-up）**

给引脚通过一个电阻接到电源 VCC，使其默认为 **高电平**。

作用：防止悬空、保持稳定逻辑状态。

**下拉（Pull-down）**

给引脚通过一个电阻接到 GND，使其默认为 **低电平**。

同样是为了防止悬空。

**弱上拉（Weak Pull-up） / 弱下拉（Weak Pull-down）**

所谓“弱”就是电阻很大（几十 kΩ \~ 上百 kΩ），只提供一个轻微的电流，不会影响其他强驱动电路。

常用于 MCU、FPGA 未使用的 IO 或输入管脚，保证逻辑稳定。

**三态（High Impedance，简称 Hi-Z）**

表示引脚处于高阻抗，不输出电流，相当于“断开”。

常用于 **输入** 或 **总线共享** 的场景。

**常见 IO 驱动模式**

**推挽输出（Push-Pull Output）**

用两个 MOS 管上下对称，一个接 VCC，一个接 GND。

可以主动输出高电平和低电平。

特点：驱动能力强，速度快。

缺点：**不能并联多个推挽输出**，否则可能短路。

应用：SPI、MCU 普通 GPIO 输出。

高电平时：上管导通 → 输出 VCC低电平时：下管导通 → 输出 GND

**开漏输出（Open-Drain，OD，或 Open-Collector）**

只有下管（接 GND），没有上管。

想输出低电平：下管导通 → 接地。

想输出高电平：下管关闭 → 需要外部 **上拉电阻** 才能拉高。

特点：可以多个 IO 并联到同一线上（“线与”），互不冲突。

应用：I²C 总线、复位信号、INT 中断线。

输出低电平：下管导通输出高电平：下管关断，靠外部上拉电阻

**三态输出（Tri-state）**

在推挽输出的基础上，加一个“使能控制”。

三种状态：高电平、低电平、高阻抗。

特点：适合总线共享。

应用：并行总线、FPGA 内部总线。

使能=0 → 输出高阻态 (Z)使能=1 → 正常推挽输出 (0/1)

**输入模式（Input）**

IO 作为接收端，内部是高阻态。

可配置是否带弱上拉或弱下拉，避免悬空。

🔑 **工程习惯：**

FPGA未使用的 IO → **配置为输入+弱上拉**。

需要总线多点驱动 → **三态或开漏**。

普通 GPIO → **推挽输出**。

I²C → **开漏输出+上拉电阻**。

**FPGA / 数字 IC 设计中的 IO 模式与未用管脚处理**

在 FPGA 工程里，如果 IO 悬空，可能导致：

输入级 CMOS 门处于亚稳态 → 功耗升高、EMI 增大。

输入噪声耦合进 FPGA → 信号完整性变差。

所以综合工具 / Bitstream 配置里通常会提供 **Unused Pin Options**，常见有：

**Pull-up（弱上拉输入）** → 常用，默认逻辑“1”，最稳定。

**Pull-down（弱下拉输入）** → 也可以，但较少用。

**High-Z（悬空）** → 一般不推荐，除非外部电路另有约束。

📌 在 Xilinx Vivado 或 Intel Quartus 里，默认选项往往就是 **“Weak Pull-up”**。

**FPGA 常见 IO 驱动模式**

**推挽输出（SSTL/LVCMOS 默认模式）**

FPGA IO 最常见的输出模式。

内部是上下对称的 MOS 管，可以主动驱动 0 和 1。

应用：大多数高速总线（DDR、并口）、普通 GPIO。

**开漏输出（Open-Drain/Open-Collector）**

FPGA IO 内部只驱动“0”，输出“1”时转为高阻态。

必须靠外部上拉电阻。

应用：I²C、共享中断信号、复位。

在 FPGA 中要么直接支持 IO 标准，要么用三态逻辑自己实现：

assign sda = drive_low ? 1'b0 : 1'bz;

**三态输出（Tri-state Buffer）**

FPGA 内部通常不允许真正的三态总线（综合工具会转成 MUX），但在 **IO 层**仍然可以使用三态，配合 IO Buffer (OBUFT)。

应用：外部总线共享，如 SDRAM、Flash 数据线。

例子：

assign io_data = oe ? data_out : 1'bz;

**差分 IO（LVDS、CML 等）**

高速接口常用，成对 IO 引脚工作。

FPGA IO Bank 要选择支持差分标准的电压。

应用：高速串行链路（SERDES、ADC/DAC LVDS 接口）。

**对比总结（FPGA 工程视角）**

| 模式        | FPGA 内部表现        | 应用场景                   |
| ----------- | -------------------- | -------------------------- |
| 输入+弱上拉 | 内部电阻上拉至 VCC   | 未使用的管脚、保证稳定     |
| 推挽输出    | OBUF，驱动 0/1       | DDR、GPIO、大部分并行接口  |
| 开漏输出    | OBUFT，仅驱动 0，1=Z | I²C、复位、INT 共享        |
| 三态输出    | OBUFT，带 OE 控制    | 外部总线共享（SRAM/Flash） |
| 差分输出    | OBUFDS（成对驱动）   | LVDS、SERDES、ADC/DAC 接口 |

**未使用的 FPGA/ASIC 管脚**，一般设置为 **输入 + 弱上拉**，避免悬空噪声。
