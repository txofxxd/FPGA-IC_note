# **语法篇**

## **摩尔型状态机和米利型状态机**

-   ### **Moore 型 FSM**

输出 **只依赖于当前状态**。

状态决定输出，输入不会直接影响输出。

输出信号稳定，变化只在状态跳转时发生（随时钟边沿变化）。

优点：设计稳定、容易时序收敛；缺点：可能需要更多状态。

也就是说你的状态机的每一个状态下只有一句输出的值不同，例：

state0：out\<=1;state1:out\<=0;这个感觉，所以可能需要更多状态

-   ### **Mealy 型 FSM**

输出 **依赖于当前状态和输入**。

输入的变化会立即影响输出（组合逻辑）。

优点：需要的状态数少；缺点：输出可能有毛刺（因为输入组合逻辑的直接作用）。

非moore基本都是mealy型，常写的状态机基本都是mealy型，就是每一个状态要干很多事情进行很多判断才会确定输出值的内容，为了防止组合逻辑的毛刺影响状态，一般都会将组合逻辑过一下DFF才会进行状态判断

## **Verilog 可综合 vs 不可综合语句总结表**

### **✅ 常见可综合语句（能转成硬件电路）**

**点击图片可查看完整电子表格**

| 语句/结构             | 说明             | 综合结果             |
| --------------------- | ---------------- | -------------------- |
| assign                | 组合逻辑连续赋值 | 逻辑门/连线          |
| always @(posedge clk) | 时序逻辑         | 寄存器/触发器        |
| always @(*)           | 组合逻辑         | 组合逻辑电路         |
| case / if-else        | 条件选择         | 译码器/多路选择器    |
| for（有界循环）       | 用于生成硬件     | 循环展开成并行硬件   |
| function              | 组合逻辑复用     | 被内联展开为逻辑电路 |
| generate              | 结构化重复生成   | 实例化多个硬件单元   |
| parameter             | 参数定义         | 常量替换，综合时展开 |

**🚫 常见不可综合语句（仅用于仿真/Testbench）**

| 语句/结构                             | 说明               | 原因                                                         |
| ------------------------------------- | ------------------ | ------------------------------------------------------------ |
| initial                               | 仿真初始块         | 综合工具不支持（少数 FPGA 支持寄存器上电初值，但不可移植）   |
| # 延时                                | 仿真时间控制       | 硬件无“延时语句”，只能用计数器实现                           |
| forever                               | 无限循环           | 电路资源有限，无法实现无限过程                               |
| while                                 | 条件不定的循环     | 无法静态展开成硬件（若条件固定，可用 for）                   |
| task                                  | 行为级复用         | 本身不可综合，内部若全是可综合语句，有的工具可内联展开，但不保证 |
| $display / $monitor / $stop / $finish | 仿真打印、结束语句 | 仅仿真相关                                                   |
| $random                               | 随机数生成         | 硬件无此机制，需要自写 LFSR/PRBS                             |
| fork-join                             | 并行仿真进程       | 综合工具不支持                                               |

**⚠️ 部分语句综合限制**

-   for：循环次数必须是**常量可确定**的（才能展开成并行硬件）。
-   while：一般不可综合，除非工具支持并且条件是常量展开。
-   task：不同综合器支持不同，一般不推荐用在可综合代码中。

📌 **记忆口诀：**

-   **组合逻辑、时序逻辑语句可综合；**
-   **仿真语句（initial, \#, \$...）不可综合；**
-   **循环要定界，函数能综合。**

### **Verilog 保留字分类表**

**1️⃣ 常用（RTL 开发中高频出现）**

这些是写 FPGA/ASIC 设计必备的：

-   **结构相关**：module, endmodule, input, output, inout, wire, reg, parameter, localparam
-   **过程语句**：always, initial, assign, begin, end, if, else, case, for, while, repeat, forever
-   **时序控制**：posedge, negedge, @(…), \#delay
-   **仿真任务/函数**：task, function, disable

**2️⃣ 中等常用（综合支持但不常考）**

-   **强弱驱动相关**：tri, tri0, tri1, wand, wor
-   **数据类型相关**：integer, time, real, realtime
-   **特殊赋值**：deassign, release
-   **宏定义控制**：define\`,\` \`include\`, ifdef

**3️⃣ 冷门（大多用于门级仿真/电气建模，笔试陷阱区⚡）**

这类你在 RTL 开发几乎不会用到：

-   **驱动强度**：supply0, supply1, strong0, strong1, pull0, pull1, weak0, weak1, highz0, highz1
-   **延迟建模**：small, medium, large
-   **强制驱动**：force, release
-   **门级描述**：nmos, pmos, cmos, tran, tranif0, tranif1, rtran, rtranif0, rtranif1
-   **电阻器件**：rnmos, rpmos, rcmos

**4️⃣ 非保留字（常作为干扰项 ❌）**

-   container, class（Verilog-95/2001 中没有；class 只在 SystemVerilog 有意义）
-   macro, object 等（不是 Verilog 关键字）

**📌 总结口诀**

-   **考试常见陷阱** → 考冷门关键字：large、force、supply0/1、tran。
-   **RTL 设计** → 基本只会用到第一类 + 部分第二类。
-   **门级/工艺建模** → 第三类才会出现，但 RTL 写代码不用。

**\`timescale time_unit/time_presicion（单位/精度）**

单位与精度的取值只能为1/10/100的整数倍，比如1.5，0.5之类的是不合法的

## **代码覆盖率**

| 覆盖率类型    | 又称 / 等级   | 检查内容                                  | 举例说明                                                     |
| ------------- | ------------- | ----------------------------------------- | ------------------------------------------------------------ |
| 语句覆盖率    | 代码 1 覆盖率 | 每一行代码是否至少执行过一次              | if (a>0) b=1; else b=0; → 检查 b=1; 和 b=0; 两行是否都跑到过 |
| 分支覆盖率    | 代码 2 覆盖率 | 判断语句的每个分支是否被执行过            | if/else、case 的各分支都要跑到                               |
| 条件覆盖率    | 代码 3 覆盖率 | 逻辑条件的每个布尔子表达式是否都取过真/假 | if (a>0 && b==1) → 要测试 a>0 真/假，b==1 真/假              |
| 路径覆盖率    | 高级覆盖率    | 判断语句组合的所有路径是否都跑到          | if-else + case → 所有组合路径                                |
| FSM 覆盖率    | 状态覆盖      | 状态机中每个状态和转换是否被激活          | FSM 所有状态、所有跳转路径都要被走到                         |
| Toggle 覆盖率 | 翻转覆盖      | 信号位是否发生过 0→1 和 1→0 翻转          | 检查关键寄存器/总线是否真正活动过                            |
| 功能覆盖率    | 功能级别覆盖  | 是否覆盖了设计需求里的功能点              | 自定义覆盖点，比如“FIFO 满/空 各触发过一次”                  |
